指令：include_directories([AFTER|BEFORE] [SYSTEM] dir1 dir2 … )
该指令用来向工程添加多个特定的头文件搜索路径，通过AFTER或者DEFORE参数来控制添加的头文件路径是放在已有路径的前面还是后面。除此之外，还可以通过set变量CMAKE_INCLUDE_DIRECTORIES_BEFORE为on来控制是追加还是置前。
指令：link_directories(dir1 dir2)
该指令用来向工程添加多个非标准的共享库搜索路径。
指令：target_link_libraries(target lib1 <debug | optimized> lib2 …)
该指令用来为target添加需要链接的共享库。
（3）注意：
a. 用命令 ldd bin/main 检查main的链接情况
b. 链接到静态库：将target_link_libraries指令修改为 target_link_libraries(main libhello.a)
c. 特殊环境变量 CMAKE_INCLUDE_PATH 和 CMAKE_LIBRARY_PATH

  1. 如果头文件没有存放在常规路径（/usr/include, /usr/local/include等），则可以通过这些变量就行弥补。例如，可以使用bash方法如下：
    export CMAKE_INCLUDE_PATH=/home/liu/usr/include/hello
    然后在头文件中将include_directories(/home/liu/usr/include/hello)替换为：
    find_path(myheader hello.h) # find_path用来在指定路径中搜索文件名
    if(myHeader)
    include_directories(${myHeader})
    endif(myHeader)
    如果不使用find_path，CMAKE_INCLUDE_PATH变量的设置是没有作用的。

  2. 同样地，库文件也可用CMAKE_LIBRARY_PATH来设置，搭配find_libraries()使用。

八、find_package命令

8.1 用法

FIND_PACKAGE( <name> [version] [EXACT] [QUIET] [NO_MODULE] [ [ REQUIRED | COMPONENTS ] [ componets… ] ] )

在Module模式中，cmake需要找到一个叫Find<LibraryName>.cmake的文件。在Conifg模式中，则为<LibraryName>Config.cmake或者<LibraryName>-config.cmake的文件。

  • version参数

    需要一个版本号，它是正在查找的包应该兼容的版本号（格式是major[.minor[.patch[.tweak]]]）。

  • EXACT选项

    要求版本号必须精确匹配。如果在find-module内部对该命令的递归调用没有给定[version]参数，那么[version]和EXACT选项会自动地从外部调用前向继承。对版本的支持目前只存在于包和包之间（详见下文）。

  • QUIET 参数：

    会禁掉包没有被发现时的警告信息。对应于Find.cmake模块中的 NAME_FIND_QUIETLY。

  • REQUIRED 参数

    其含义是指是否是工程必须的，表示如果报没有找到的话，cmake的过程会终止，并输出警告信息。对应于Find<name>.cmake模块中的 NAME_FIND_REQUIRED 变量。

  • COMPONENTS参数

    在REQUIRED选项之后，或者如果没有指定REQUIRED选项但是指定了COMPONENTS选项，在它们的后面可以列出一些与包相关（依赖）的部件清单（components list）
